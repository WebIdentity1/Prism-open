# Self-Hosting Prism Salon OS

This guide is for salon owners, agencies, and developers who want to run Prism with their own infrastructure and provider accounts.

If an AI agent is doing the setup for a salon owner, start with [AI_AGENT_SETUP.md](./AI_AGENT_SETUP.md). It includes the credential interview script, provider-by-provider secret collection checklist, safer command patterns, validation steps, and a no-secret deployment handoff template.

## 1. Prerequisites

- Node.js 20.19 or newer
- npm
- Supabase CLI
- A Supabase project, or local Supabase for development
- Vercel CLI or another static hosting provider if deploying the frontend
- Provider accounts for the optional integrations you want to enable

If an AI agent is doing the setup, authorize access before asking it to deploy:

- Supabase CLI login or Supabase connector/plugin access to the target organization and project.
- Vercel CLI login, Vercel MCP, or equivalent deployment-provider access to the target team/project.
- GitHub/repository access if the agent will push changes or connect Git-based deploys.
- Provider dashboard access or owner availability for Stripe, Gemini, ElevenLabs, Twilio, Resend, Google Cloud, DNS, and billing/compliance approvals.

The agent should run the mechanical setup itself after authorization. You should only need to step in for browser login, 2FA, paid plan approval, DNS ownership, live payments/SMS compliance, and provider keys that must be created in a dashboard.

Install dependencies:

```sh
npm install
```

Create your env file:

```sh
cp .env.example .env
```

Run the setup doctor:

```sh
npm run setup:doctor
```

## 2. Required Supabase Setup

Create a Supabase project and fill these values in `.env`:

```sh
VITE_SUPABASE_URL=
VITE_SUPABASE_PUBLISHABLE_KEY=
SUPABASE_URL=
SUPABASE_ANON_KEY=
SUPABASE_PUBLISHABLE_KEY=
SUPABASE_SERVICE_ROLE_KEY=
PUBLIC_SITE_URL=
SITE_URL=
```

For local development, `PUBLIC_SITE_URL` and `SITE_URL` can be:

```sh
http://localhost:8080
```

Apply database migrations to your Supabase project:

```sh
supabase login
supabase link --project-ref <your-project-ref>
supabase db push
```

For local Supabase development:

```sh
supabase start
supabase db reset
```

This repo uses non-default local Supabase ports to avoid collisions with other projects:

```text
API:    http://127.0.0.1:54341
DB:     postgresql://postgres:postgres@127.0.0.1:54342/postgres
Studio: http://127.0.0.1:54343
Mail:   http://127.0.0.1:54344
```

After `supabase start`, run `supabase status` and copy the local publishable key into:

```sh
VITE_SUPABASE_URL=http://127.0.0.1:54341
VITE_SUPABASE_PUBLISHABLE_KEY=<local-publishable-key>
```

## 3. Supabase Auth URLs

In Supabase Auth settings, configure:

- Site URL: your deployed frontend URL
- Redirect URLs:
  - `http://localhost:8080/*`
  - `https://your-domain.example/*`

## 4. Edge Function Secrets

Set secrets for the provider integrations you want to enable:

```sh
supabase secrets set GEMINI_API_KEY=...
supabase secrets set STRIPE_SECRET_KEY=...
supabase secrets set STRIPE_WEBHOOK_SECRET=...
supabase secrets set ELEVENLABS_API_KEY=...
supabase secrets set VOICE_AGENT_SECRET=...
supabase secrets set TWILIO_ACCOUNT_SID=...
supabase secrets set TWILIO_AUTH_TOKEN=...
supabase secrets set TWILIO_PHONE_NUMBER=...
supabase secrets set RESEND_API_KEY=...
supabase secrets set EMAIL_FROM="Your Salon <appointments@example.com>"
supabase secrets set GOOGLE_BP_CLIENT_ID=...
supabase secrets set GOOGLE_BP_CLIENT_SECRET=...
supabase secrets set PUBLIC_SITE_URL=https://your-domain.example
supabase secrets set SITE_URL=https://your-domain.example
```

Do not set `ENABLE_DEMO_SEEDING=true` in production. Demo seeding creates known demo accounts and is intended only for local sandboxes.

Deploy functions:

```sh
supabase functions deploy
```

## 5. Optional Provider Setup

### Gemini AI

Required key:

```sh
GEMINI_API_KEY=
```

Enables:

- AI assistant
- Virtual try-on
- Face-shape detection
- Style image generation
- Style classification
- Email generation

### Stripe

Required keys:

```sh
STRIPE_SECRET_KEY=
STRIPE_WEBHOOK_SECRET=
```

Configure a webhook endpoint in Stripe:

```text
https://<your-supabase-project>.supabase.co/functions/v1/stripe-webhook
```

Enable events used by your payment flow, including checkout session, payment intent, invoice, subscription, and account updates if using Connect.

### ElevenLabs Voice Agent

Required keys:

```sh
ELEVENLABS_API_KEY=
VOICE_AGENT_SECRET=
```

`VOICE_AGENT_SECRET` should be a random value generated by you. It is used to protect tool-callback requests from the voice agent.

### Twilio

Required keys:

```sh
TWILIO_ACCOUNT_SID=
TWILIO_AUTH_TOKEN=
TWILIO_PHONE_NUMBER=
```

Enables SMS delivery and direct Twilio phone-number attachment for the voice-agent flow.

Before sending real SMS, adapt [SMS_COMPLIANCE_TEMPLATE.md](./SMS_COMPLIANCE_TEMPLATE.md) for the deployment owner and confirm consent, STOP/HELP behavior, privacy-policy language, and any regional messaging requirements.

### Resend

Required keys:

```sh
RESEND_API_KEY=
EMAIL_FROM=
```

Use a verified sending domain before sending real salon email.

### Google Business Profile

Required keys:

```sh
GOOGLE_BP_CLIENT_ID=
GOOGLE_BP_CLIENT_SECRET=
```

Configure the OAuth callback URL:

```text
https://<your-supabase-project>.supabase.co/functions/v1/google-bp-auth
```

### Demo Mode

Demo login and sample data are optional and off by default. For local demos only:

```sh
VITE_ENABLE_DEMO_LOGIN=true
supabase secrets set ENABLE_DEMO_SEEDING=true
```

The demo seed functions create known-password users for client, stylist, and salon-owner walkthroughs. Leave both flags disabled for production and for any deployment containing real client data.

## 6. Frontend Deployment

Build the app:

```sh
npm run build
```

Deploy the generated `dist/` directory to Vercel, Netlify, Cloudflare Pages, or any static host that supports single-page app rewrites.

For Vercel, this repo includes `vercel.json` with a fallback rewrite to `index.html`.

Vercel CLI flow:

```sh
npx vercel@latest login
npx vercel@latest whoami
npx vercel@latest link
npx vercel@latest env add VITE_SUPABASE_URL production
npx vercel@latest env add VITE_SUPABASE_PUBLISHABLE_KEY production
npx vercel@latest env add VITE_SITE_URL production
npx vercel@latest env add VITE_ENABLE_DEMO_LOGIN production
npx vercel@latest --prod
```

Use Vercel only for browser-safe frontend variables in this app. Supabase service-role keys, provider API keys, webhook secrets, and OAuth client secrets belong in Supabase Edge Function secrets.

After the final frontend URL is known, update Supabase:

```sh
supabase secrets set PUBLIC_SITE_URL=https://your-domain.example SITE_URL=https://your-domain.example
```

Then update Supabase Auth URLs, Stripe webhooks, Google OAuth redirects, and DNS/custom-domain settings to match the final frontend and function URLs.

## 7. Verification

Before using the app with real clients:

```sh
npm run setup:doctor
npm run build
npm run test
```

Then verify manually:

- Owner signup and salon onboarding
- Staff invite
- Client booking
- Client onboarding link
- Payment checkout if Stripe is enabled
- SMS if Twilio is enabled
- Email if Resend is enabled
- AI consultation if Gemini is enabled
- Voice agent if ElevenLabs is enabled

## 8. Production Notes

- Use separate development and production Supabase projects.
- Keep service-role keys out of the browser.
- Rotate keys after accidental exposure.
- Leave demo login and demo seeding disabled in production.
- Review RLS policies before storing real client records.
- Confirm SMS, email, privacy, payment, tax, and retention requirements for your jurisdiction.
